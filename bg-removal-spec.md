# 製造業向け背景除去ツール 仕様書（ブラウザ内完結版）

## 1. 概要・目的

製造業の現場画像（部品、治具、検査対象物）に特化した背景除去ツール。
最大の差別化ポイントは「画像が一切サーバーに送信されない」こと（社外秘・機密部品写真への対応）。
推論からマスク修正、出力まで全てブラウザ内（クライアントサイド）で完結させる。

既存のCloud Run版とは別パターンとして、完全クライアントサイド実装を行う。

## 2. 技術前提

- 推論: ブラウザ内ONNX実行（`onnxruntime-web` または `@imgly/background-removal`）
  - WebGPU対応時はWebGPU backend、非対応時はWASMにフォールバック
- フロントエンド: React + Canvas API（react-konvaも可、要件次第）
- ネットワーク通信: 画像データに関しては一切発生させない（モデルファイルの初回ダウンロードのみ許容）
- PWA化を見据えた構成（Service Workerでモデルキャッシュ）が望ましい

## 3. 全体フロー

1. 画像アップロード（ドラッグ&ドロップ / ファイル選択 / 複数枚バッチ対応）
2. ブラウザ内推論で背景除去 → アルファマスク生成
3. プレビュー表示（元画像 + マスク適用後画像）
4. 修正ツールでマスクを編集
5. 透明PNG（または指定背景色）でエクスポート

## 4. 機能仕様

### 4.1 背景除去（推論）

- 入力画像 → モデル推論 → 0-255のアルファマスク（confidence map）を生成
- アルファマスクは元画像と同サイズのグレースケールデータとして保持（後続の修正処理で再利用するため、二値化する前のconfidence値を保持すること）

### 4.2 マスク修正ツール（本仕様の核心）

ブラシ操作は採用しない。誤り方のパターン別に専用ツールを用意する。

#### 4.2.1 クリックトグル（Connected Component方式）— 優先度: 高

- マスクに対して連結成分（Connected Component Labeling）を事前計算し、「島」単位でグループ化する
- ユーザーが画像上の任意の点をクリックすると、その点が属する島全体を選択
- 選択した島を「透明化」または「不透明化」にワンクリックで反転
- 用途: 誤って残った背景の小片、誤って消えた部品の穴や凹部の復元
- 実装: flood fill / BFS等によるラベリング。1クリックごとに該当ラベル領域のアルファ値を一括反転

#### 4.2.2 範囲選択 + 一括反転 — 優先度: 高

- 矩形選択、または投げ縄（自由形状）選択
- 選択範囲内を「全て前景にする」「全て背景にする」のどちらかに一括変更
- 用途: 直線的なエッジ（治具のフレーム、台座など）のまとめ修正

#### 4.2.3 ローカルしきい値再調整 — 優先度: 中

- 元のconfidence map（4.1で保持した値）に対し、選択範囲内だけしきい値スライダーで再二値化
- 用途: 金属反射などで境界が滲んだ箇所の締め直し
- スライダー操作のみで完結。選択範囲外には影響を与えない

#### 4.2.4（将来拡張・Pro機能候補）ポイントプロンプト式再セグメンテーション

- SAM系モデル（ONNX化したもの）をブラウザ内ロードし、「ここは前景」「ここは背景」のポイントクリックで再推論
- 初期実装では対象外。モデルサイズとロード時間がボトルネックになるため、需要が見えてから着手

### 4.3 表示・UX要件

- ズーム/パン機能（小部品の境界を精密に扱うため必須）
- Undo/Redo（操作ごとにマスクのスナップショットをスタックで保持）
- 境界線の薄いオーバーレイ表示（推論直後のマスク境界をガイドとして表示、ON/OFF切り替え可）
- 操作対象（島・選択範囲）のハイライト表示

### 4.4 出力

- 透明PNGエクスポート（単体・バッチZIP）
- 背景色指定エクスポート（検査基準のグレー等、製造業向けプリセットを用意）
- 入力時のファイル名・命名規則を維持してエクスポート

## 5. データモデル（実装の目安）

```text
ImageState {
  originalImageData: ImageData        // 元画像
  confidenceMap: Uint8ClampedArray    // 推論結果の生confidence値（0-255、二値化前）
  alphaMask: Uint8ClampedArray        // 現在の表示用マスク（編集結果反映後）
  labelMap: Int32Array                // Connected ComponentのラベルID（クリックトグル用に再計算が必要なタイミングで更新）
  history: AlphaMaskSnapshot[]        // Undo/Redo用スナップショット
}
```

- `labelMap` はマスクが変更されるたびに再計算が必要（範囲選択や閾値変更後の島の再分割に対応するため）。毎クリックごとの全体再計算はコストが高い場合、変更領域のみ局所再ラベリングする最適化を検討

## 6. 非機能要件

- 画像データのネットワーク送信は一切行わない（DevToolsのNetworkタブで検証可能なレベルで保証）
- 推論・修正操作はオフライン環境（Wi-Fi切断状態）でも動作すること
- 一般的なノートPCで、1枚あたりの推論+マスク表示が数秒以内に完了すること

## 7. 受け入れ基準（チェックリスト）

- [ ] オフライン状態（ネットワーク切断）でアップロード→背景除去→修正→出力が完了する
- [ ] クリックトグルで孤立した島を1クリックで透明化/復元できる
- [ ] 矩形/投げ縄選択で範囲を一括反転できる
- [ ] しきい値スライダーで選択範囲内のみ再二値化できる
- [ ] Undo/Redoが操作単位で機能する
- [ ] ズーム状態でも修正操作の座標がズレない
- [ ] バッチ処理時、元のファイル名規則を維持して出力される
- [ ] 透明PNGと指定背景色PNGの両方をエクスポートできる
